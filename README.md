# 小红书活动随机文案 H5 模板

这是一个纯前端静态网页，不需要数据库或服务器程序。适合线下活动二维码：

1. 扫码打开网页；
2. 随机显示一条文案，并自动拼接固定话题；
3. 用户点击按钮复制；
4. 用户打开小红书，选择照片、粘贴并发布。

## 你需要修改的内容

用文本编辑器打开 `index.html`，搜索 `const CONFIG`：

- `fixedHashtags`：改成活动固定话题；
- `profileUrl`：在小红书账号主页点击“分享-复制链接”，粘贴进来；
- `topicUrl`：在活动话题页点击“分享-复制链接”，粘贴进来；
- `copyPool`：把示例文案替换为你的 100 句文案。

每句文案的格式为：

```js
"这是第一句文案。",
"这是第二句文案。",
```

最后一句保留或删除逗号都可以。

## 本地预览

直接双击 `index.html` 可以看页面。但“复制到剪贴板”在部分浏览器中只允许 HTTPS 网站使用，因此正式测试应先部署上线。

## 上线方式

### 快速测试：GitHub Pages

把 `index.html` 上传到 GitHub 仓库，进入 Settings → Pages，选择从 main 分支发布。

### 国内正式活动：腾讯云 CloudBase 静态网站托管

在腾讯云 CloudBase 创建静态网站应用，上传整个文件夹。正式活动建议使用公司已有备案域名或由公司技术人员协助绑定域名。

## 生成二维码

网站上线后，复制网页 URL，用任意二维码生成器生成二维码并打印。二维码内容必须是网页 URL，不是本地文件路径。

## 上线前务必测试

至少用以下环境实测：

- iPhone 微信扫一扫；
- 安卓微信扫一扫；
- 小红书 App 扫码；
- 手机系统相机扫码；
- 4G/5G 网络，不只测试公司 Wi‑Fi。

重点检查：网页打开速度、复制按钮、账号链接、话题链接、中文话题是否完整。

## 能做与不能做

能做：随机文案、固定话题、一键复制、跳转账号/话题分享链接、统计按钮点击（需另接统计服务）。

不能保证：自动关注、自动选图、自动粘贴、自动发布；普通网页也不应依赖未公开的小红书私有 URL Scheme。
